Fluxos do Apex

1. Contrato scenario.yaml

Define o anti-pattern, dados e sinais esperados. É a única ligação entre os geradores.

2. Gerador de código

python3 code_generator.py scenario.yaml job.py → produz job PySpark com bug na linha especificada.

3. Gerador de plano (sintético)

python3 plan_generator.py scenario.yaml synthetic.ndjson → gera event log sem rodar Spark, em milissegundos.

4. Execução real

docker exec spark-master spark-submit job.py → log real armazenado no MinIO.

5. Validação

Script Python compara os sinais esperados com o log sintético. Exemplo de saída: join_operator: SortMergeJoin e recordsRead: 200100.

6. Oráculo (planejado)

Workflow semanal que baixa logs reais do MinIO, compara com sintéticos e abre issue em caso de divergência.

7. CI gate (planejado)

Em cada PR, rodar plan_generator para todos os YAMLs e validar o acceptance; bloquear merge se falhar.
